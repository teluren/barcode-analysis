# Выводы

## 🔬 Что удалось установить

### Общая структура

Билеты на электричку используют **CODE-128** — линейный штрихкод. В более новых билетах (№1 и №2) — **два** CODE-128 по 18 цифр. В старом (№3) — **один** CODE-128 со встроенным разделителем.

### Формат 18-значных билетов (№1 и №2)

```
Баркод A: 22 XX XX 45 01 XX XX XX 33
Баркод B: 00 XX XX XX XX XX XX XX 11
```

| Часть | Позиции | Значение | Статус |
|-------|---------|----------|--------|
| `22` | 0-1 | Префикс баркода A | ✅ Фиксирован |
| `45 01` | 6-9 | Код маршрута / зоны / тарифа | ✅ Фиксирован |
| `33` | 16-17 | Постфикс баркода A | ✅ Фиксирован |
| `00` | 0-1 | Префикс баркода B | ✅ Фиксирован |
| `11` | 16-17 | Постфикс баркода B | ✅ Фиксирован |
| Поз. 5, 8, 10 в B | — | Значения `4`, `5`, `8` | ✅ Совпадают в обоих |
| Поз. 2-3 в A | 2-3 | **Кодирует направление** | 🔄 `85` (туда) → `39` (обратно) |
| Поз. 5 в A | 5 | **Кодирует направление** | 🔄 `9` (туда) → `7` (обратно) |
| Поз. 10-15 в A | 10-15 | Уникальный номер билета | ❌ Полностью разный |
| Баркод B целиком | — | Уникальный ID транзакции | ❌ Полностью разный |

### Что кодирует направление

При сравнении билета **туда** (№1) и **обратно** (№2) в баркоде A стабильно меняются:

- **Позиции 2-3**: `85` (Александровская→Балтийский) → `39` (Балтийский→Александровская)
- **Позиция 5**: `9` → `7`

Это, скорее всего, **код станции отправления** или **битовый флаг направления** в структуре билета.

### Формат старого билета (№3)

```
05114375*1716
```

Другой формат — вероятно, другой терминал продажи или версия ПО автомата.  
`*` выступает разделителем между кодом билета и контрольной суммой.

### Баркод B (00XX..11)

Баркод B полностью разный у всех билетов. Это **уникальный идентификатор покупки/транзакции**:
- Префикс `00` — возможно, тип операции
- Постфикс `11` — возможно, индикатор успешной операции
- Внутренние цифры — уникальный ID сессии

## 📚 Результаты исследования (13.07.2026)

Проведён поиск информации о формате штрихкодов CODE-128 на билетах РЖД.

### Известный формат АСУ «Экспресс-3» (бумажные билеты, 14 цифр)

На бумажных билетах, оформляемых через терминалы АСУ «Экспресс-3», штрихкод включает:

1. **Номер дороги** (2 цифры) — код железной дороги, оформившей билет
2. **Номер пункта продажи** (3 цифры)
3. **Дата отправления** (3 цифры, порядковый номер дня в году, Julian)
4. **Номер поезда** (5 цифр): первые 3 — номер поезда, последние 2 — буквенное обозначение нитки
5. **Контрольная цифра**

Всего: 2 + 3 + 3 + 5 + 1 = 14 цифр.

Источник: [Bilet.ru](https://www.bilet.ru/rus/railstat-blankcis.htm), [Aviashop.ru](https://www.aviashop.ru/bilety-na-poezd/rasshifrovka-shtrih-koda-na-bilete/)

### Отличия нашего формата (18 цифр)

Наши 18-значные коды **не вписываются** в 14-значную схему АСУ «Экспресс-3». Это говорит о том, что:
- Электронные/мобильные билеты используют **расширенный формат**
- Возможно, добавлены поля: ID транзакции, время, тип билета
- Приложение — скорее всего **«РЖД Пассажирам»** или **«Пригород»** (оператор **АО «СЗППК»** — Северо-Западная пригородная пассажирская компания)

### Ключевые находки

| Источник | Что нашли |
|----------|-----------|
| АСУ «Экспресс-3» документация | Формат 14 цифр: дорога(2)+пункт(3)+дата(3)+поезд(5)+КЦ(1) |
| СЗППК / РЖД моб. приложения | Динамический штрихкод, меняется при каждом открытии/покупке |
| СПб станции | Балтийский вокзал — ЕСР код **035300**, экспортный код присвоен приказом Росжелдора №449 от 13.11.2012 |

### Прикладная методология: как декодировать неизвестный формат

1. **Сравнительный анализ** — ✅ сделано (3 билета, выявлен шаблон)
2. **Статистический анализ** — нужна выборка ≥10 билетов
3. **Изоляция переменных** — менять один параметр за раз:
   - Купить 2 билета подряд (одинаковый маршрут, время) → увидеть счётчик
   - Купить на разные даты → найти поле даты
   - Купить на разные станции → найти поле станции
4. **Контрольная сумма** — проверить, подчиняется ли последняя цифра Luhn/CRC
5. **Cross-reference** — сравнить с открытыми спецификациями GS1-128 / IATA / ISO 15417

## ❓ Что осталось неизвестным

| Вопрос | Почему | Как решить |
|--------|--------|-----------|
| Как закодирована дата поездки? | Нет двух билетов с одной датой | Купить 2 билета на разное время одного дня |
| Как закодировано время? | Нет данных | Сравнить последовательные покупки |
| Как закодирована цена/тариф? | Все билеты по одному тарифу | Купить билет на другой маршрут |
| Код станции в структуре? | Мало станций | Купить билет на другую станцию |
| Точный протокол кодирования | Закрытый формат РЖД | Анализ по методике выше |
| Почему билет №3 — другого формата? | Другой терминал/оператор | Сравнить с билетами из того же автомата |
